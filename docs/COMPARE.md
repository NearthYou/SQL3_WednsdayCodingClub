# 비교 분석

비교 대상은 별도 프로젝트가 지정되지 않았으므로, 이 저장소의 기존 CLI-only baseline과 현재 API+MVCC 버전을 비교합니다.

## 한 줄 요약

- baseline: “엔진 내부 구현의 재미”가 강한 CLI형 미니 DBMS
- current: “엔진 + API + 동시성 + 시연성”까지 확장한 포트폴리오형 미니 DBMS 서버

## 주요 특징 비교

| 항목 | 기존 CLI-only baseline | 현재 API+MVCC |
| --- | --- | --- |
| 외부 사용 방식 | SQL 파일 입력 | REST API + CLI 병행 |
| 동시성 제어 | 사실상 단일 프로세스/단일 실행 흐름 중심 | table-snapshot COW MVCC |
| 트랜잭션 | 명확한 API 트랜잭션 없음 | `/tx`, rollback, conflict detect |
| 병렬 처리 시연 | 제한적 | API pool + DB pool + `/page` trace |
| 내부 지표 | 로그 중심 | `/metrics` endpoint |
| 테스트 체계 | 시나리오/벤치 중심 | unit + API + CI |
| 설명/발표성 | 엔진 설명에 강함 | 엔진 + 시스템 설계 설명에 강함 |

## 현재 버전의 장점

- 외부 클라이언트가 실제 HTTP로 접근 가능해서 활용성과 데모성이 크게 올라갔습니다.
- snapshot 일관성을 갖는 `/page` 데모가 있어 “하나의 요청이 여러 SQL로 병렬 분해된다”는 메시지가 분명합니다.
- 트랜잭션 실패 시 rollback이 눈에 보이는 형태로 동작합니다.
- `/metrics` 덕분에 thread pool 과 MVCC 상태를 바로 설명할 수 있습니다.
- `make test` 와 CI까지 연결되어 제출/포트폴리오 완성도가 높습니다.

## 현재 버전의 단점

- MVCC가 row-level이 아니라 table-snapshot 기반이라, 같은 테이블에 대한 동시 write는 보수적으로 충돌 처리됩니다.
- legacy CLI 엔진과 신규 API 엔진이 완전히 하나의 코어로 합쳐진 것은 아니라서, 장기적으로는 코드 중복 정리가 필요합니다.
- API는 CRUD만 노출하므로 DDL/스키마 관리까지는 아직 가지 않았습니다.
- durability는 full WAL이 아니라 CSV flush 수준이라, “진짜 DBMS” 관점에서는 한 단계 더 남아 있습니다.

## baseline의 장점

- 구조가 단순해서 이해와 디버깅이 쉽습니다.
- delta log, `.idx` snapshot, row/page cache 같은 기존 엔진 요소를 직관적으로 보기 좋습니다.
- 단일 실행 경로에서는 오버헤드가 적고 빠르게 실험하기 좋습니다.

## baseline의 단점

- 외부 시스템과 연결되는 사용성은 거의 없습니다.
- 병렬성, tx, 격리 수준 같은 서버 관점 설명이 약합니다.
- 결과가 stdout 중심이라 자동화와 통합이 어렵습니다.

## 점수표

프로젝트 목표를 “DBMS 포트폴리오 + 시연성 + 시스템 설계 설명력” 기준으로 잡고 10점 만점으로 평가했습니다.

| 평가 항목 | baseline | current |
| --- | ---: | ---: |
| 엔진 구조의 흥미도 | 8 | 9 |
| 외부 사용성 | 3 | 9 |
| 동시성/격리 설명력 | 2 | 7 |
| 트랜잭션 안정성 | 3 | 8 |
| 테스트/품질 체계 | 4 | 8 |
| 발표/시연 적합성 | 5 | 10 |
| 운영 단순성 | 9 | 6 |
| 합계 | 34 / 70 | 57 / 70 |

## 총평

기존 baseline은 “작은 엔진을 직접 만들었다”는 인상을 주는 데 강합니다. 현재 버전은 거기에 API, MVCC, 병렬 처리, rollback, metrics, CI를 더해 “시스템으로 확장할 줄 안다”는 메시지까지 얹습니다.

포트폴리오 기준으로는 현재 버전의 점수가 더 높습니다. 다만 순수 엔진 구현의 밀도와 단순함은 baseline 쪽이 더 깔끔하므로, 발표에서는 두 버전을 “엔진 단계”와 “서비스 단계”로 나눠 설명하는 것이 가장 설득력 있습니다.
